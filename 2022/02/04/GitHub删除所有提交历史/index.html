GitHub删除所有提交历史

Created2022-02-04|Updated2022-02-04
|Post View:

效果

将代码保持在当前状态，清空commit history。

思路

创建新分支 -> 将文件添加到新分支 -> 提交更改 -> 删除旧master分支 -> 重命名新分支 -> 强制更新仓库

步骤

  1. 创建新分支

  2. 将文件添加到新分支

  3. 提交更改

  4. 删除旧master分支

  5. 重命名新分支

  6. 强制更新仓库
